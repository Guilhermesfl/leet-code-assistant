# Database Modeling

## Overview
Database modeling is the process of designing the structure of a database to store and retrieve data efficiently.

## Key Concepts
- Normalization and denormalization
- Indexing
- Query patterns
- Relational vs NoSQL databases

## Patterns & Techniques
- ER diagrams
- Star and snowflake schemas
- Read/write optimization
- Data partitioning

## Example Interview Questions
- How would you model a social network database?
- What is normalization and why is it important?
- How do indexes improve query performance?

## Practice
- Design a schema for an e-commerce site
- Optimize queries for large datasets
